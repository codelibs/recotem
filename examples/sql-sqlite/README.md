# SQLite SQL data source example

Zero external services. Trains a recommender from a local SQLite database using
the `sql` data source.

## Files

- `seed.py` — creates `events.db` with 50 synthetic interaction rows (5 users × 10 items)
- `recipe.yaml` — SQL recipe pointing at `RECOTEM_RECIPE_DB_DSN`
- `events.db` — generated by `seed.py`; listed in `.gitignore`

## Run

From the repository root:

```bash
# 1. Seed the database (creates examples/sql-sqlite/events.db).
uv run python examples/sql-sqlite/seed.py

# 2. Generate a signing key and export it.
#    keygen prints four lines; grep out just the env_entry line.
export $(uv run recotem keygen --type signing | grep '^env_entry=' | sed 's/^env_entry=/RECOTEM_SIGNING_KEYS=/')

# 3. Point Recotem at the SQLite file.
#    The DSN lives in an env var — never in the recipe itself.
export RECOTEM_RECIPE_DB_DSN="sqlite:///$(pwd)/examples/sql-sqlite/events.db"

# 4. Train.
mkdir -p artifacts
uv run recotem train examples/sql-sqlite/recipe.yaml
```

The artifact is written to `./artifacts/sql_sqlite_demo.recotem`.

## What it demonstrates

- A `sql` data source with a parameterised query (`WHERE event_at >= :since`).
- DSN supplied via `RECOTEM_RECIPE_DB_DSN` (the `RECOTEM_RECIPE_*` allow-list);
  credentials never appear in the recipe file.
- TopPop and RP3beta compared via Optuna recall@5.
- `always_overwrite` versioning — suitable for local iteration.

## Adapting to PostgreSQL or MySQL

Change only the DSN; the recipe stays the same:

```bash
export RECOTEM_RECIPE_DB_DSN="postgresql+psycopg://user:pass@localhost/mydb"
# or
export RECOTEM_RECIPE_DB_DSN="mysql+pymysql://user:pass@localhost/mydb"
```

Install the matching extra first:

```bash
uv add 'recotem[postgres]'   # psycopg3
uv add 'recotem[mysql]'      # PyMySQL
```
